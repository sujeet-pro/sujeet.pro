# Content Structure

Guidelines for all content on sujeet.pro. Read this before creating, reviewing, or updating any content.

## Target Audience

Staff/Principal/Senior engineers, IT management (Director/VP), interview prep candidates. Content should be technically deep, well-structured, and suitable as a reference.

## Content Types

### Articles (`content/articles/<slug>/README.md`)

Deep-dive technical content. Grouped into **series** for related topics.

**Required frontmatter:**

```yaml
---
title: "Article Title"
description: "One-line description for SEO meta tags and listing cards"
publishedDate: 2026-03-15
lastUpdatedOn: 2026-03-20
tags: [distributed-systems, cap-theorem]
---
```

**Optional:** `draft: true` to exclude from build.

**Do NOT set** `layout` or `category` in frontmatter — layouts are resolved from `meta.json5`.

**Structure rules:**

- Start with an H1 title (the only H1 on the page)
- Follow with an abstract/intro paragraph summarizing the core mental model
- Use H2/H3 hierarchy (never skip levels — no H2 → H4)
- Include diagrams for architecture, flows, decision trees
- Use tables for comparisons
- Use GitHub Alerts for important notes, warnings, tips
- End with practical takeaways or summary
- All factual statements must have citations (inline links or footnotes)

**Title/description are NOT rendered in the article.** They are only used for SEO meta tags, RSS feed, and listing cards. The markdown's own H1 is the visible title.

### Blogs (`content/blogs/<slug>/README.md`)

Shorter-form, personal/exploratory content. Auto-ordered by `publishedDate` (newest first).

**Required frontmatter:** Same as articles.

**Differences from articles:**

- More personal tone, opinion pieces acceptable
- Research depth is lighter (not every claim needs a citation)
- No series assignment
- Shorter, more focused

### Projects (`content/projects/<slug>/README.md`)

Technical project descriptions with architecture and outcomes.

**Required frontmatter:** Same as articles.

**Additional notes:**

- Include architecture overview diagram
- Link to repo, demo, etc.
- Focus on technical decisions and outcomes

### Listing Pages (`content/articles/README.md`, `content/blogs/README.md`)

Index pages for each content type. Rendered with the `Listing` layout.

**Frontmatter:** `title`, `description`, `tags` (optional).

### Home Page (`content/README.md`)

Site homepage. Rendered with the `Home` layout.

**Frontmatter:** `title`, `description`, `layout: Home`.

## Series

A **series** is a named group of related articles. Defined in `content/articles/meta.json5`:

```json5
{
  series: [
    {
      slug: "distributed-systems",
      displayName: "Distributed Systems",
      shortName: "DS",
      description: "Fundamentals of distributed systems",
      articles: ["cap-theorem", "consistency-models", "consensus-protocols"],
    },
  ],
}
```

Articles within a series get prev/next navigation in the left sidebar.

## Directory Layout

```
content/<type>/<slug>/
  README.md               # Content (required)
  diagrams/               # Diagram source files (.mermaid, .excalidraw, .drawio, .dot)
    flow.mermaid
    flow-light.svg         # Generated by diagramkit (gitignored)
    flow-dark.svg          # Generated by diagramkit (gitignored)
  assets/                 # Images and other assets
    screenshot.png
```

## Layout Resolution

Layouts are resolved from `meta.json5`, not frontmatter:

| Content Type | Item Layout           | Listing Layout |
| ------------ | --------------------- | -------------- |
| Articles     | Article               | Listing        |
| Blogs        | Blog                  | Listing        |
| Projects     | Project               | Listing        |
| Home         | Home                  | —              |
| Tags         | TagIndex / TagListing | —              |

## Responsive Layout

- **Desktop (≥140ch):** 3 columns — left sidebar (series nav), content, right sidebar (TOC)
- **Tablet (110ch–140ch):** 2 columns — content + TOC. Left sidebar → hamburger overlay.
- **Mobile (<110ch):** 1 column. TOC → collapsible accordion above content.
- Home/listing/tag pages: no left sidebar

## Citation Requirements (Articles)

Every factual statement must be supported by a reference. Authority hierarchy:

1. Official specifications (ECMAScript, RFCs, W3C, WHATWG)
2. Official documentation (MDN, language docs, framework docs)
3. Academic papers (peer-reviewed)
4. Recognized experts (core contributors, TC39 members, language designers)
5. Well-known technical blogs (with track record of accuracy)

**If a claim cannot be fact-checked, do not include it.**

### Citation Formats

Inline: `According to the [ECMAScript specification](https://tc39.es/ecma262/), ...`

Footnote:

```md
The event loop processes microtasks before macrotasks[^1].

[^1]: [WHATWG HTML Living Standard — Event Loops](https://html.spec.whatwg.org/multipage/webappapis.html#event-loops)
```
